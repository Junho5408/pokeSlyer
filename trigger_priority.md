# [시스템 명세] 이벤트 트리거 및 실행 우선순위

## 1. 트리거 포인트 (Trigger Points)
시스템은 전투 중 특정 시점에 이벤트를 발행하며, 유물이나 카드는 이에 반응한다.

| 트리거 종류 | 발생 시점 | 대표 사례 |
| :--- | :--- | :--- |
| **OnTurnStart** | 플레이어 턴이 시작될 때 | 에너지 회복, 독 데미지 정산 |
| **OnCardPlay** | 카드가 손패에서 필드로 나갈 때 | 카드 사용 시 유물 스택 증가 |
| **OnAttack** | 공격 판정이 발생할 때 | 힘 버프 데미지 추가 |
| **OnDamageTaken** | 유닛이 피해를 입었을 때 | 가시 데미지 반격, 방어도 삭감 |
| **OnKill** | 적의 체력이 0이 되었을 때 | 적 처치 시 체력 회복 유물 |
| **OnTurnEnd** | 턴 종료 버튼을 눌렀을 때 | 방어도 초기화, 턴 종료 효과 |

## 2. 실행 우선순위 (Priority Level)
동일한 트리거에 여러 효과가 동시에 반응할 경우, 아래 순서에 따라 처리한다.

1. **시스템 레벨 (System)**: 턴 제한, 강제 이벤트 등 최우선 처리.
2. **지속 상태 효과 (Status)**: 독, 화상, 재생 등 상태 이상 정산.
3. **유물 레벨 (Relic)**: 보유 중인 유물의 패시브 효과.
4. **카드 레벨 (Card)**: 실제 사용한 카드의 고유 로직.

## 3. 액션 큐 (Action Queue) 처리 원칙
- 모든 액션은 발행 순서대로 큐(Queue)에 담겨 순차 실행된다.
- **Interrupt(끼어들기)**: 특정 액션 실행 중에 파생된 액션(예: 반격)은 현재 액션 바로 뒤에 삽입되어 먼저 처리된다.
- 연출(애니메이션)은 큐의 실행과 동기화되어야 하며, 로직 처리가 끝날 때까지 다음 입력을 제한한다.

## 4. 예외 처리
- 유닛이 죽는 순간 모든 예약된 액션 큐에서 해당 유닛을 타겟팅하는 액션은 무효화(Cancel) 처리한다.
